# Agent Rules

- Kapanpun Anda diminta untuk melakukan git commit, hindari penggunaan awalan "feat:", "fix:", "chore:", atau awalan konvensional sejenisnya secara default (kecuali jika diminta secara eksplisit). Tulis pesan commit secara langsung, jelas, dan menggunakan bahasa yang diminta oleh pengguna (misalnya: "menambahkan fitur unduh pdf", "perbaikan typo").
- Selalu tulis kode (terutama frontend) dengan cara yang sederhana, rapi, dan mudah dipahami oleh pemula. Hindari penggunaan logika atau arsitektur yang terlalu kompleks/ribet. Fokus pada hasil visual yang menarik dan modern dengan menambahkan animasi, transisi, atau efek hover (Tailwind CSS) agar tampilan UI terlihat keren.
